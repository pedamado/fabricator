# Computational Design Letterpress Fabricator

**Live Tool:** [http://typedesign.fba.up.pt/letterpress-fabricator/](http://typedesign.fba.up.pt/letterpress-fabricator/)

A browser-only tool that turns a user-supplied **variable font** (`.ttf` / `.otf`), **SVG vector artwork**, or **bitmap image** into 3D-printable letterpress sorts (slugs) ready for SLA or FDM — *and now* into paired **matrix + counter-matrix dies** for inkless **embossing & debossing**. Drop a file, click a glyph, hit export. The geometry comes out at standardized typographic heights ready to mix with the 19th-century type sitting in the next composing stick over.

Designed for the FBAUP Type Design course so students with no 3D background can take their digital type — and now their logotypes, icons, engravings, and embossed bookplates — straight to the school's fablab and print on the workshop's letterpress presses.

## Description

This app streamlines the fabrication of custom variable fonts and one-off graphics designed digitally in the Type Design course of the Master in Graphic Design and Editorial Projects at FBAUP. It is aimed at students with no prior knowledge of 3D modelling software — drop a font (or an SVG, or even a photo of a sketch), click a few glyphs (or just the artwork), export STL.

The geometry is standardized to the same typographic measurements as the workshop's existing metal and wood type, so the printed sorts mix straight back into the Printmaking Workshop's composing sticks and presses.

## Features

- **Variable Font Parsing.** Drop any TrueType (`.ttf`) or OpenType (`.otf`) variable font — the app reads `fvar`, axes, named instances and outlines directly in the browser via `opentype.js` and a bundled `samsa-core` ES module. Variation-aware geometry: every change to a font axis (wdth / wght / etc.) or to a named instance rebuilds both the printed eye *and* the underlying slug body from the live variated outline.
- **NEW · Graphic Sources (v9.0).** Two additional dropzones sit under the font dropzone:
  - **Vector dropzone** — drop an `.svg` file (logotype, icon, illustration). The canvas is treated as a 1000-unit EM square; bounding box, scale, and Y-flip are handled automatically so the artwork lands on the typographic baseline. (PDF / AI ship in v9.1 — export as SVG from Illustrator for now.)
  - **Bitmap dropzone** — drop any common image (JPG, PNG, TIFF, HEIF, WebP, GIF). The image is traced through `ImageTracer.js` with a forced 2-colour palette and the resulting contours feed the same builder pipeline. **Tip:** use maximum-contrast black-and-white images (≥ 600 × 600 px) for the cleanest trace.
  - All sources flow through one builder, so every feature below applies whether you started from a font, a vector, or a bitmap.
- **True letterpress metrics.** Sorts are output at a configurable typographic height with presets for Continental (23.56 mm), Anglo-American (20.32 mm shoulder), Italian (21.60 mm) and Dutch (21.85 mm), so they mix with the existing FBAUP workshop type.
- **Parametric Sort Generation.** Solid slugs by default, switching to hollow shells with internal walls plus SLA drain hatches once a sort passes the minimum-width threshold. Every sort gets foot chamfers, a foot ID-nick, a baseline marker nick (where appropriate) and a 12° reinforced neck (slope) on the relief.
- **★ Embossing & Debossing pairs (v9.1, refined in v9.1.3).** Forget ink — make **paper itself** carry the shape. A second accordion turns any source (font glyph, SVG, or traced bitmap) into a matched **matrix + counter-matrix** sandwich that physically reshapes a sheet between them:
  - **Two modes.** Choose *Embossing* (the print rises off the page) or *Debossing* (it sinks into it). The matrix and counter automatically swap roles between convex die and concave cavity.
  - **Press-ready or hand-tool ready.** Default geometry keeps the slug at 20.56 mm + 0.6 mm relief so the pair stacks to the press's typographic travel — or flip on **Hand-embosser mode** and both halves drop to a 1 mm + 0.6 mm pair sized for plier-style desk tools (the 40 mm round default matches a stationer's emboss-stapler).
  - **Paper tolerance baked in.** A single slider (0.025–1.00 mm) sets the cavity-side offset that gives the paper room to crease without binding. In-panel guidance: 0.05–0.15 mm for 80–120 gsm letter stock, 0.15–0.30 mm for cardstock, 0.30+ for cover board / leather.
  - **Inverted-slope cavity walls** flare outward toward the opening so paper folds in cleanly without snagging — the same trick traditional die-makers use, here computed automatically from a configurable 0–45° angle.
  - **Plate shape: glyph-bounds, rectangle, or ellipse.** Lock-aspect ratio with a checkbox; independently scale the glyph from 0.25× to 4× inside the plate. Mix initials, ornaments, and full alphabets at will.
  - **Counter is X-mirrored opposite to the matrix** so when the student flips it onto the platen the two halves register perfectly — no math required on your end.
  - **Export the pair.** Single STL holds both meshes; batch ZIP emits `<glyph>_matrix.stl` + `<glyph>_counter.stl` inside an `_Embossing` or `_Debossing` folder, so the print queue never gets confused.
- **Internal Support Walls for large sorts (v8.1).** When a hollow cavity exceeds 4 × wall thickness, the cavity is automatically subdivided into a grid of cells with internal vertical and horizontal support walls. Each cell stays within a configurable maximum span (15 mm by default), and each row / column gets its own drain hatch — preventing unsupported spans across the top of the letter on tall point-sizes (SLA sag, FDM bridging failure).
- **Advanced Customization Panel.** A full second-tier accordion exposes every slug parameter: font size (10–288 didot pt), type-height presets and overrides, hollow toggle, wall thickness, support-wall toggle and cell span, drain toggle and diameter, foot chamfer size, foot-nick and baseline-nick radii, beard tolerance (% of EM, per side), and reinforced-neck slope angle. Every change is persisted across the session in `sessionStorage`, and a single "Reset Advanced Defaults" button restores the workshop preset.
- **Bulk glyph selection.** Filter glyphs by category (Uppercase / Lowercase / Figures / Punctuation / Spacing Quads) or type a custom string. Select all / Clear buttons handle bulk selection. (Hidden automatically when a vector / bitmap source is active.)
- **Direct-to-Plate Layouts.** Free floating layout or predefined physical plates (20 × 20 cm, 9 × 12 cm) with automatic row wrapping.
- **Exporting.** Single or bulk export via high-fidelity ASCII STL, Wavefront OBJ, or batch ZIP archives with one STL per glyph (or a single STL bundle for a graphic source).

## How it works (technical summary)

- No bundler — everything runs as ES modules from `<script type="module">`.
- `three@0.160.0` + `three-csg-ts@3.1.11` drive the boolean geometry for the printing-matrix path: each slug body is built as a Box, then individual cells, drains and nicks are subtracted via CSG.
- The **embossing cavity** is built differently — `three-csg-ts` produced non-manifold output on curved glyph subtractions, so the cavity-bearing plate is now a single `THREE.ExtrudeGeometry` of a `THREE.Shape` whose `.holes[]` array contains the glyph contours. Slope direction comes from `bevelSize`; uniform paper-tolerance offset from `bevelOffset`. Earcut handles perimeter-with-holes reliably even for letters with counters (O, B, e, a, …). The slug/base layer underneath is a separate mesh in the same group — slicers merge the two on touch.
- A unified `GlyphSource` contract (see `js/sources/glyph-source.js`) sits between the parsers and the builder, so the same engine handles font glyphs, SVG paths and bitmap-traced contours. The builder's public entry point is `buildSlugFromSource(source, mirror, applyDraft, variableSize, slugOptions)`; the legacy `buildBlock(...)` is a thin font-specific shim.
- `samsa-core` (vendored locally under `src/samsa-core.js`) is the source of truth for variable-instance glyph outlines, because `opentype.js`'s `gvar` support is unreliable for variation outlines. The samsa instance is tag-keyed (`{wght: 800, wdth: 75}`) and the resulting `SamsaGlyph` is reused both for bounds calculation and for shape extraction so the slug width/height always tracks the currently variated outline.
- `opentype.js@1.3.4` is retained for font metrics, glyph-list discovery and a fallback outline path for non-variable fonts.
- `imagetracerjs@1.2.6` (CDN, UMD) powers the bitmap-tracing pipeline with a 2-colour palette; the resulting SVG is filtered to drop the white "paper" layer and parsed by the in-house SVG parser.
- `jszip@3.10.1` (UMD) packages the batch export.

## Changelog

- **v9.1.5 (May 2026)** — **Remove-overlap for variable-font glyphs.** Vendored [`polygon-clipping`](https://github.com/mfogel/polygon-clipping) (Mike Fogel's Martinez-algorithm port — pure ESM, ~75 KB with two transitive deps `splaytree` + `robust-predicates`). The cavity pipeline now runs a polygon-union pass between `ShapePath.toShapes()` and `ExtrudeGeometry`, flattening overlapping or self-intersecting sub-contours into clean disjoint shapes. Fixes the "loop-corner step" visible in heavy-weight `2`, `5`, `6`, `g` cavities. Same approach the old Processing/Geomerative library used.
- **v9.1.4 (May 2026)** — **Cavity construction rebuilt for complex glyphs.** Replaced the "all subpaths as plate holes" approach (which broke on composite variable-font glyphs at extreme weights — `e`, `g`, `a` cavities failing or rendering empty) with `THREE.ShapePath.toShapes(false)` + counter islands. Each glyph shape's outer outline becomes a plate hole; each counter becomes a separate solid-island mesh inside the cavity. Inherits the same robustness as the raised-relief side. New viewport overlay: **☀ Bright lighting** toggle (adds three extra directional lights + boosted ambient so shallow embossing cavities are easy to read) and **◐ X-ray** toggle (translucent rendering for inspecting interior geometry).
- **v9.1.3 (May 2026)** — Counter is always X-mirrored opposite to the matrix, so when the student flips the counter face-down onto the platen the two halves register correctly. Independent layouts and contour-path sets per half.
- **v9.1.2 (May 2026)** — **Cavity construction rebuilt.** Replaced the CSG cavity subtraction (which was producing non-manifold output on curved/composite glyphs — missing base faces, inverted walls) with a single `THREE.ExtrudeGeometry` of a `THREE.Shape` whose `.holes[]` are the glyph contours. Slope via `bevelSize`, paper tolerance via `bevelOffset`. CSG is now restricted to chamfers + foot nick + baseline nick on the slug body. Default relief retuned to **0.6 mm** (range 0.3–2.0 mm) with paper-weight guidance in-panel.
- **v9.1.1 (May 2026)** — Fixed Z-mirror winding in `makeReliefExtrudeGeometry` so the inverted-slope relief renders with outward-facing normals (the v9.1.0 release rendered as a "hollow box" because every triangle was back-facing after the Z-flip).
- **v9.1 (May 2026)** — **Embossing & Debossing module.** A new advanced-panel accordion generates a paired **matrix + counter-matrix** from any source (font glyph, SVG, bitmap). Modes: Debossing (lowered into paper) and Embossing (raised on paper). Heights preserve the press's typographic travel (slug 20.56 mm + 1 mm relief on the matrix; 1 mm base + 1 mm relief on the counter). The cavity side of the pair gets a uniform **paper-tolerance offset** (0.025–1.00 mm slider; 0.10 mm default for 80 gsm). The relief sides use an **inverted slope** (exact at the deep end, flaring outward at the configured angle) so paper can crease without snagging. Optional **hand-embosser mode** drops the slug entirely for plier-style tools. Plate shape can be glyph bounds (auto), rectangular, or round/ellipse; glyph scale is independently controllable (0.25× – 4×). Implementation reuses the v9.0 `GlyphSource` contract and the v8.1 CSG plumbing — no new external dependencies. Batch ZIP emits `<name>_matrix.stl` and `<name>_counter.stl` per glyph.

- **v9.0.1 (May 2026)** — All external CDN dependencies vendored under `vendor/` (three, three-csg-ts, opentype.js, jszip, imagetracerjs, Inter typeface). The `three-csg-ts` +esm bundle's hard-pinned `three@0.143.0` import was rewritten at vendor time to point at the local `three@0.160.0`. Fully offline-capable — drop the project folder onto a USB stick and it still works.

- **v9.0 (May 2026)** — New Graphic Source pipelines: drop an `.svg` vector file or any bitmap image (`.jpg`/`.png`/`.heif`/`.tif`/`.webp`/`.gif`) and the same builder produces a letterpress sort. SVG parsing is direct (custom path parser, full d-attribute grammar, CTM flattening, Y-flip, height-normalized to 1000 EM units). Bitmaps are traced via `ImageTracer.js` with a forced 2-colour palette and the white background filtered out. The builder was refactored around a modular `GlyphSource` contract (`js/sources/`) so the font, vector, and bitmap pipelines all feed one engine. Two new dropzones in the sidebar; font-only panels (Instance / Axes / Glyph Selection) auto-hide when a graphic source is active. PDF / AI parsing ships next in v9.1.
- **v8.1 (May 2026)** — Internal support walls for large hollow sorts. When a cavity exceeds 4 × wall thickness, it is subdivided into an MxN cell grid (each cell ≤ 15 mm by default, configurable) with one drain hatch per row and per column. New advanced controls: Internal Support Walls toggle + Max cell span slider.
- **v8 (May 2026)** — Advanced Customization accordion with full slug-shape parameter set; `sessionStorage` persistence; Reset Advanced Defaults button; Select all / Clear bulk-selection buttons.
- **v7.1 (May 2026)** — Fixed variable-instance outline pipeline (samsa wired as ES module, axis-tag-keyed `sf.instance(...)`, instance glyph reused for both bounds and shape; slug width/height now derived from the variated outline's actual on-curve + Bézier-control bounds).
